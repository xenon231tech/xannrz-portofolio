# Accessibility Guidelines

## Our Commitment

XANNRZ Portfolio is committed to ensuring digital accessibility for people with disabilities. We strive to conform to WCAG 2.1 Level AA standards and continuously improve the accessibility of our website and services.

## Accessibility Standards

### WCAG 2.1 Compliance

We follow the Web Content Accessibility Guidelines (WCAG) 2.1, which provide recommendations for making web content more accessible to people with disabilities. Our target is Level AA compliance.

**The four principles of WCAG are:**

1. **Perceivable** - Information must be perceivable to users
2. **Operable** - Users must be able to operate the interface
3. **Understandable** - Content must be understandable
4. **Robust** - Content must be robust enough for interpretation by assistive technologies

## Accessibility Features

### Visual Accessibility

- **Color Contrast** - Text has sufficient contrast ratio (4.5:1 for normal text, 3:1 for large text)
- **Resizable Text** - Users can resize text up to 200% without loss of functionality
- **No Color Dependency** - Information is not conveyed by color alone
- **Focus Indicators** - Visible focus indicators for keyboard navigation
- **Responsive Design** - Works on all screen sizes and orientations

### Keyboard Navigation

- **Tab Navigation** - All interactive elements are accessible via keyboard
- **Skip Links** - Skip to main content links available
- **Keyboard Shortcuts** - Common shortcuts documented and accessible
- **No Keyboard Trap** - Users can navigate away from any element
- **Focus Order** - Logical and intuitive focus order

### Screen Reader Support

- **Semantic HTML** - Proper use of HTML elements
- **ARIA Labels** - Descriptive labels for interactive elements
- **Alt Text** - Descriptive alt text for all images
- **Form Labels** - Associated labels for form inputs
- **Headings** - Proper heading hierarchy

### Audio and Video

- **Captions** - All videos include captions
- **Transcripts** - Audio content has transcripts
- **Audio Descriptions** - Important visual information is described
- **Controls** - Accessible media player controls
- **No Auto-play** - Media doesn't auto-play

### Forms and Input

- **Clear Labels** - All form fields have clear labels
- **Error Messages** - Clear and helpful error messages
- **Required Fields** - Clearly marked as required
- **Input Validation** - Helpful validation messages
- **Accessible Inputs** - Support for various input methods

## Testing and Validation

- **Zoom Testing** - Test at 200% zoom level
